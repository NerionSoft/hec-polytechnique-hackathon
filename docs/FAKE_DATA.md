# Fake data inventory

Source-of-truth document. Lists every piece of fake / mock / hardcoded data still living in the codebase, where it sits, why it is there, and what replaces it once a given module is wired to the real back-end.

Rule of thumb (set 2026-04-26):

- **Production code paths must not depend on fake data.** Lead sourcing, enrichment, scoring, outreach all run on real APIs (Sirene, Cheerio scrape, Vercel AI Gateway / Gemini) or persisted DB rows.
- **Fake data is allowed only inside `src/application/test-support/`** (consumed by `*.test.ts` and `*.integration.test.ts`).
- The only production-side exception is **outreach email sending**: no SMTP integration, the "send" button flips a `SENT` flag locally for the demo. The draft body itself is generated by the real LLM. This is documented under _Outreach_ below.
- Anything still in `src/lib/mock/` is **legacy landing-period UI scaffolding** for routes outside the lead → outreach scope (pipeline deal detail, IC, audit, team). It must be replaced or the route deleted as soon as that route is brought into scope.

## 3. Test-only fakes (allowed)

`src/application/test-support/` — used by `vitest --project unit`. Never imported from `src/app/`, `src/presentation/`, or production `src/infrastructure/` code (enforced by ESLint `no-restricted-imports`).

- `fakes/InMemory*Repository.ts` — port doubles for unit tests.
- `fakes/Fake{Clock,IdGenerator,CompanyDataSource,WebsiteScraper,LeadEnricher}.ts` — port doubles for unit tests.
- `builders/{LeadBuilder,FundThesisBuilder,EnrichmentResultBuilder}.ts` — fluent fixtures for tests only.

Any future port (e.g. `OutreachDrafter`) gets an in-memory fake here, **and only here**.

## 4. Outreach — no SMTP integration

Hackathon time-box: no Resend / SendGrid / Gmail wiring. The pipeline is real up to the draft, then fakes the send:

- The email body is **really** generated by `OutreachDrafter` calling Gemini via the AI Gateway, using the persisted enrichment + thesis as input. The body is not fake.
- The "Send" button calls `PATCH /api/outreach/[id] { status: "SENT" }`. No SMTP call is made. The lead's status is also flipped to `CONTACTED`.
- The UI offers **Copy** (clipboard) and **Open in mail client** (`mailto:` link prefilled with subject + body) so the user can actually send the email by hand.
- The draft, model used, prompt hash, generation timestamp, and approval timestamp are all persisted, so the audit trail tells the truth: "draft generated by `google/gemini-3-flash` at HH:MM, marked sent by user at HH:MM, no automated SMTP delivery occurred."

If Resend gets wired later, add an `OutreachSender` port. Until then, the contract is honest about what happened.

## 5. Maintenance protocol

- **A new file under `src/lib/mock/` must not be added.** If you find yourself reaching for one, the work is out of scope and belongs in a separate ticket.
- **A new test fixture goes under `src/application/test-support/`,** never alongside production code.
- **When a production module replaces a mock**, delete the corresponding mock file in the same PR and update the table above.
- **When a route is deleted because its mock is gone and there is no real data yet**, prefer deletion over an empty-state placeholder fed by mocks.
